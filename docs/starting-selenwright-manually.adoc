== Starting Selenwright Manually

[NOTE]
====
. This guide shows how to start Selenwright manually.
. This guide assumes you already know how to deal with command line and know basic Docker commands.
====

=== Prepare Browsers

There are two ways to populate the browser catalog:

==== Option A: Browser Discovery (recommended)

Pull the browser images you need and let Selenwright discover them automatically:

----
$ docker pull selenwright/playwright-chromium:latest
$ docker pull selenwright/playwright-firefox:latest
$ docker pull selenwright/playwright-webkit:latest
----

Start Selenwright — it will detect the pulled images via Docker labels. Then open <<Browser Discovery, Selenwright UI>> and **adopt** the discovered images into the catalog. No configuration file needed.

==== Option B: Manual `browsers.json`

Create `config/browsers.json` configuration file:

.config/browsers.json
[source,json]
----
{
    "chromium": {
        "default": "1.56.1",
        "versions": {
            "1.56.1": {
                "image": "selenwright/playwright-chromium:latest",
                "port": "3000",
                "path": "/",
                "protocol": "playwright"
            }
        }
    },
    "firefox": {
        "default": "1.56.1",
        "versions": {
            "1.56.1": {
                "image": "selenwright/playwright-firefox:latest",
                "port": "3000",
                "path": "/",
                "protocol": "playwright"
            }
        }
    }
}
----

NOTE: Playwright images expose port `3000` at path `/` and require `"protocol": "playwright"`. For classic Selenium images use `"path": "/"` for Chrome/Opera and `"path": "/wd/hub"` for Firefox. See <<Browsers Configuration File>> for all fields.

Pull the images:

----
$ docker pull selenwright/playwright-chromium:latest
$ docker pull selenwright/playwright-firefox:latest
----

Selenwright falls back to this file when no adopted images are found.

==== Browser Images

Selenwright publishes Playwright-based browser images on Docker Hub:

* https://hub.docker.com/r/selenwright/playwright-chromium[`selenwright/playwright-chromium`]
* https://hub.docker.com/r/selenwright/playwright-firefox[`selenwright/playwright-firefox`]
* https://hub.docker.com/r/selenwright/playwright-webkit[`selenwright/playwright-webkit`]

Each image carries `io.selenwright.*` Docker labels so <<Browser Discovery>> picks them up without a configuration file. Dockerfiles live in https://github.com/aqa-alex/selenwright-browsers[aqa-alex/selenwright-browsers].

Classic Selenium-protocol images are not published under `selenwright/` — bring your own (for example, upstream `selenoid/chrome`, `selenoid/firefox`).

[IMPORTANT]
====
Browser Discovery and `browsers.json` are mutually exclusive: as soon as Selenwright finds any adopted image, it builds the catalog from Docker labels and ignores `browsers.json` entirely. To mix Playwright and classic Selenium browsers, pick one mechanism and put *all* entries there:

* **JSON-only** — don't adopt anything in the UI; list every browser (Playwright and Selenium) in `browsers.json`.
* **Discovery-only** — tag your Selenium images with the required `io.selenwright.*` labels (see https://github.com/aqa-alex/selenwright-browsers[selenwright-browsers] for the label contract) and adopt them alongside the Playwright images.
====

=== Start Selenwright
==== Option 1: start Selenwright binary

. Download binary for your operating system from https://github.com/aqa-alex/selenwright/releases/latest[releases] page
and save it as `selenwright` (or `selenwright.exe` on windows).
+
NOTE: Add execution permission in case of *nix os-type with `chmod +x selenwright`.

. Then run:
+
.*nix
----
./selenwright
----
+
.Windows
----
selenwright.exe
----

. It should write to console something like:
+
----
2017/11/26 21:23:43 Loading configuration files...
2017/11/26 21:23:43 Loaded configuration from [config/browsers.json]

...

2017/11/26 21:23:43 Listening on :4444
----

==== Option 2: start Selenwright container

If you have https://docs.docker.com/engine/installation/[Docker] installed you can omit downloading binary and run it inside container.
Pull browser images, then run:

.*nix
[source,bash,subs="attributes+"]
----
docker run -d                                   \
--name selenwright                                 \
-p 4444:4444                                    \
-v /var/run/docker.sock:/var/run/docker.sock    \
selenwright/hub:latest-release
----

TIP: If you use a manual `browsers.json` (Option B above), mount it as a volume: `-v /your/directory/config/:/etc/selenwright/:ro` and add `-conf /etc/selenwright/browsers.json` to the command.
