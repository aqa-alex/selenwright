== Starting Selenwright Manually

[NOTE]
====
. This guide shows how to start Selenwright manually.
. This guide assumes you already know how to deal with command line and know basic Docker commands.
====

=== Prepare Browsers

There are two ways to populate the browser catalog:

==== Option A: Browser Discovery (recommended)

Pull the browser images you need and let Selenwright discover them automatically:

----
$ docker pull selenwright/chrome:130.0
$ docker pull selenwright/firefox:131.0
----

Start Selenwright — it will detect the pulled images via Docker labels. Then open <<Browser Discovery, Selenwright UI>> and **adopt** the discovered images into the catalog. No configuration file needed.

==== Option B: Manual `browsers.json`

Create `config/browsers.json` configuration file:

.config/browsers.json
[source,json]
----
{
    "chrome": {
        "default": "130.0",
        "versions": {
            "130.0": {
                "image": "selenwright/chrome:130.0",
                "port": "4444",
                "path": "/"
            }
        }
    },
    "firefox": {
        "default": "131.0",
        "versions": {
            "131.0": {
                "image": "selenwright/firefox:131.0",
                "port": "4444",
                "path": "/wd/hub"
            }
        }
    }
}
----

NOTE: For Chrome and Opera images `path` is `"/"`. For Firefox images `path` is `"/wd/hub"`. For Playwright images set `"protocol": "playwright"` — see <<Browsers Configuration File>> for all fields.

Pull the images:

----
$ docker pull selenwright/chrome:130.0
$ docker pull selenwright/firefox:131.0
----

Selenwright falls back to this file when no adopted images are found.

=== Start Selenwright
==== Option 1: start Selenwright binary

. Download binary for your operating system from https://github.com/aqa-alex/selenwright/releases/latest[releases] page
and save it as `selenwright` (or `selenwright.exe` on windows).
+
NOTE: Add execution permission in case of *nix os-type with `chmod +x selenwright`.

. Then run:
+
.*nix
----
./selenwright
----
+
.Windows
----
selenwright.exe
----

. It should write to console something like:
+
----
2017/11/26 21:23:43 Loading configuration files...
2017/11/26 21:23:43 Loaded configuration from [config/browsers.json]

...

2017/11/26 21:23:43 Listening on :4444
----

==== Option 2: start Selenwright container

If you have https://docs.docker.com/engine/installation/[Docker] installed you can omit downloading binary and run it inside container.
Pull browser images, then run:

.*nix
[source,bash,subs="attributes+"]
----
docker run -d                                   \
--name selenwright                                 \
-p 4444:4444                                    \
-v /var/run/docker.sock:/var/run/docker.sock    \
aqa-alex/selenwright:latest-release
----

TIP: If you use a manual `browsers.json` (Option B above), mount it as a volume: `-v /your/directory/config/:/etc/selenwright/:ro` and add `-conf /etc/selenwright/browsers.json` to the command.
